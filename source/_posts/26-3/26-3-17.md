## 学习内容
### Mini-batch 梯度下降
训练过程中所有m个样本，即为一个batch。当m很大时，一次性对所有样本进行向量化运算，速度会很慢，所以我们将样本分为若干个子集，每个子集称为一个mini-batch，比如只包含1024个样本。

假如我们将样本总共分为了T个batch，则对于一个epoch（可以理解为一次完整的训练过程），大概算法流程如下：

```
for  t=1,⋯,T {
    Forward Propagation
    ComputeCostFunction
    BackwardPropagation
    W:=W−α⋅dW
    b:=b−α⋅db
}
```

对于Mini-batch梯度下降，多次epoch训练的话，每次最好将训练数据打乱并重新分组，有利于训练出更好的模型。

对于一般的模型，使用batch梯度下降，随着迭代次数的增加，损失是不断减小的，但是mini-batch梯度下降则是整体下降但有震荡。对于每组的样本数量，有两种极端情况：
- Batch gradient descent:即整个训练样本为一个batch，会平稳地接近最小值，但速度较慢；
- Stachastic gradient descent:即每组batch只有一个样本，震荡很大，难以真正达到最小值出，不能使用向量化加速运算，但前进速度很快。

实际应用中，选择一个合适的mini-batch size很重要，结合上述两种极端情况的优点。一般，当m<2000时，不分组；m很大时，每组可分为64、128、256、512等（都为2的幂）。

### 动量梯度下降算法
现在开始介绍一写常见的加快梯度下降过程的算法，首先就是动量梯度下降算法，其主要思想就是:**对梯度进行指数加权平均处理，然后用得到的梯度值更新权重W和常数项b。** 这样当前的梯度会和之前的梯度有关系，从而让梯度前进方向更加平滑，减小振荡，更快地到达最小值。

动量梯度下降算法如下，其包含了两个指数加权平均表达式：
```
# 初始时，Vdw=0,Vdb=0；一般取β=0.9
On iteration t:
    Compute dW, db on the current mini−batch
    VdW=βVdw+(1−β)dW
    Vdb=βVdb+(1−β)db
    W=W−αVdw, b=b−αVdb
```
